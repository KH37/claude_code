# PR作成コマンド仕様

以下の手順で実行してください。
## 実行フロー

### 1. ブランチ確認・作成
- **現在のブランチがmainの場合**: mainから新しいブランチを作成
	- ブランチを作成するときは、以下のルールを適用してほしい
		▼ Feature ブランチ: `feature/*`
		- **用途**: 新機能開発
		- **命名規則**: `feature/<機能名>`
		- **作成元**: `main`
		- **マージ先**: `main`
		- **特徴**: 大規模な機能開発では、このブランチをベースに複数のサブPRを作成可能

		▼ Fix ブランチ: `fix/*`
		- **用途**: バグ修正
		- **命名規則**: `fix/<修正内容>`
		- **作成元**: `main` または `feature/*`
		- **マージ先**: 作成元のブランチ

		▼ Chore ブランチ: `chore/*`
		- **用途**: ビルド、設定、依存関係更新など、機能に直接影響しない作業
		- **命名規則**: `chore/<作業内容>`
		- **作成元**: `main`
		- **マージ先**: `main`
		
		▼ Docs ブランチ: `docs/*`
		- **用途**: ドキュメント更新
		- **命名規則**: `docs/<ドキュメント内容>`
		- **作成元**: `main`
		- **マージ先**: `main`
		- **サンプルPR**:
		
		▼ Refactor ブランチ: `refactor/*`
		- **用途**: コードリファクタリング
		- **命名規則**: `refactor/<リファクタリング対象>`
		- **作成元**: `main` または `feature/*`
		- **マージ先**: 作成元のブランチ
 - **現在のブランチがmain以外の場合**: そのまま使用

### 2. コミット・プッシュ
- 変更内容のdiffを分析して適切なコミットメッセージを自動生成
- コミットメッセージは日本語で記述
- 変更をコミットしてプッシュ

### 3. PR作成
- GitHubでPRを作成
- **タイトル**: 変更内容を簡潔に表現（日本語）
- **概要**: 変更の詳細説明（日本語）

### 4. PR設定
- **アサイニー**: `KH37` を設定

### 5. 結果表示
- 作成されたPRのURLを表示
